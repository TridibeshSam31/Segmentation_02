## Overview

This project trains a **Mask2Former** semantic segmentation model on synthetic desert imagery generated by Duality AI's **Falcon** digital twin platform. The model is trained to segment 10 terrain and vegetation classes, then evaluated on an unseen desert environment to test generalization.

**Key goals:**
- Accurately segment desert environment images into 10 semantic classes
- Demonstrate model robustness on novel, unseen desert locations
- Optimize for accuracy (Mean IoU) and inference speed (<50ms/image)

---

## Model Architecture

All four model iterations use **Mask2Former** with a **Swin-Base** transformer backbone, fine-tuned for 10-class semantic segmentation.

| Parameter | Value |
|---|---|
| Architecture | `Mask2FormerForUniversalSegmentation` |
| Backbone | Swin Transformer (Base) |
| Input Resolution | 512 × 512 |
| Number of Classes | 10 |
| Decoder Layers | 10 |
| Hidden Dim | 256 |
| Queries | 100 |
| Loss Weights | Class: 2.0 · Dice: 5.0 · Mask: 5.0 |
| Image Normalization | Mean `[0.485, 0.456, 0.406]` · Std `[0.229, 0.224, 0.225]` |

## Environment Setup

### Windows

```bash
# Navigate to the ENV_SETUP folder
cd Offroad_Segmentation_Scripts/ENV_SETUP

# Run the setup script (creates a conda env called 'EDU')
setup_env.bat
```

This runs two steps automatically:
1. `create_env.bat` — creates a Python 3.10 conda environment named `EDU`
2. `install_packages.bat` — installs PyTorch (CUDA 11.8), torchvision, ultralytics, opencv, and tqdm

### Mac / Linux

Create an equivalent shell script manually:

```bash
conda create --name EDU python=3.10 -y
conda activate EDU
conda install -c pytorch -c nvidia -c conda-forge pytorch torchvision pytorch-cuda=11.8 ultralytics -y
pip install opencv-contrib-python tqdm transformers torchmetrics pandas seaborn matplotlib
```

### Activate the Environment

```bash
conda activate EDU
```

## Training Workflow

The project uses an **iterative fine-tuning strategy** across four model scripts, each building on the previous best checkpoint.

### Model 01 — Baseline Training (`road_segmentation_01.py`)

Cold-start training from `facebook/mask2former-swin-base-coco-panoptic` for 5 epochs.

```bash
conda activate EDU
cd Offroad_Segmentation_Scripts
python road_segmentation_01.py
```

**Update these paths in the script before running:**
```python
TRAIN_IMG  = r"path/to/train/Color_Images"
TRAIN_MASK = r"path/to/train/Segmentation"
VAL_IMG    = r"path/to/val/Color_Images"
VAL_MASK   = r"path/to/val/Segmentation"
```

Saves best checkpoint to: `best_offroad_model_01/`

---

### Model 02 — Fine-tuning with Augmentation (`road_segmentation_02.py`)

Loads `best_offroad_model_01`, adds **horizontal flip augmentation**, applies **class-weighted loss**, and uses a **CosineAnnealingLR** scheduler.

```bash
python road_segmentation_02.py
```

Saves best checkpoint to: `best_offroad_model_refined/`

---

### Model 03 — Oversampling for Rare Classes (`road_segmentation_03.py`)

Loads `best_offroad_model_refined`, adds **WeightedRandomSampler** to oversample images containing rare classes (Logs 5×, Ground Clutter 3×, Rocks 4×, Dry Bushes 2×). Also adds affine transforms, color jitter, and Gaussian blur augmentation.

```bash
python road_segmentation_03.py
```

Saves best checkpoint to: `best_offroad_model_refined/` (overwritten if improved)

---

### Model 04 — TTA + Diagnostics (`road_segmentation_04.py`)

Loads `best_offroad_model_01`, adds **Test-Time Augmentation (TTA)** during validation (original + flipped + brightness-adjusted), random resized crops, and per-class confusion diagnostics for Ground Clutter.

```bash
python road_segmentation_04.py
```

**Augmentation techniques applied across training:**

| Technique | Purpose |
|---|---|
| Horizontal Flip | Left/right viewpoint variation |
| Affine Transform (±15°, scale 0.85–1.15) | Simulates UGV suspension flex and dune slopes |
| Color Jitter (brightness, contrast, saturation) | Desert lighting variation |
| Gaussian Blur (kernel 3×3) | Engine vibration / dust simulation |
| Random Resized Crop (scale 0.5–1.0) | Multi-scale scene understanding |
| Test-Time Augmentation (TTA) | Ensemble predictions at inference |

## Reproducing Results

To reproduce the final best model from scratch:

```bash
# 1. Setup environment
cd Offroad_Segmentation_Scripts/ENV_SETUP
setup_env.bat

# 2. Activate environment
conda activate EDU

# 3. Run baseline training (saves best_offroad_model_01/)
cd ..
python road_segmentation_01.py   # epochs=5, batch_size=2

# 4. Fine-tune with augmentation (saves best_offroad_model_refined/)
python road_segmentation_02.py   # epochs=3, batch_size=2

# 5. Refine with oversampling
python road_segmentation_03.py   # epochs=3, batch_size=2

# 6. Final evaluation
python test.py
```

**Expected outputs:** `test_results.csv` and `confusion_matrix.png`

> Each training run saves checkpoints only when Mean IoU improves over the previous best. All metrics are logged per epoch to `training_metrics.csv`.

## Troubleshooting

**`setup_env.bat` not working on Mac/Linux?**  
Create a `setup_env.sh` with equivalent `conda` and `pip install` commands (see Environment Setup above).

**Training is too slow?**  
- Reduce `batch_size` from `2` to `1` in the script
- Close background applications to free RAM/VRAM
- Monitor GPU with `nvidia-smi` (requires CUDA-compatible GPU)

**Out of GPU memory (CUDA OOM)?**  
- Set `batch_size=1`
- Reduce image resolution in `preprocessor_config.json` from `512` to `384`

**Conda environment not found?**  
Run `conda activate EDU` from an Anaconda Prompt, not a standard terminal.

**Data management / sharing with teammates?**  
Use Google Drive, Dropbox, OneDrive, or Git LFS to share model checkpoints (`.pth`/`.bin` files are gitignored by default).
